## 1-1. OpenClawが変えるもの

「AIを使ったことはある。でも、結局コピペして自分で貼り付けているだけだ」——そんな経験をしたことはないだろうか。

ChatGPTやClaudeを使って文章を書かせ、そのテキストをコピーして別のツールに貼り付ける。要約を生成して、結果をメールに転記する。これはこれで便利だが、作業の主体は依然として人間だ。AIが「補助ツール」として使われているに過ぎない。

OpenClawが目指すのは、そこから一段階先の世界だ。

「メールをチェックして、重要なものだけまとめてSlackに送っておいて」
「毎朝8時に競合他社のサイトを確認して、価格変動があれば知らせて」
「このプロジェクトの進捗をGitHubから取得して、週次レポートにまとめて上司に送って」

こういった指示を一度しておけば、あとはエージェントが自律的に動く。人間が毎回操作する必要はない。

これがOpenClawの本質だ。**AIを「道具」から「仕事をする存在」へと変える。**

---

> 💡 **ポイント:** OpenClawは「AIと話す」ためのツールではなく、「AIに仕事をさせる」ためのプラットフォームだ。この違いを最初に理解しておくと、後の章がスムーズに読める。

---

### 1-1-1. 「エージェント」という概念

エージェント（Agent）という言葉は、AIの文脈では「自律的に行動できるAIシステム」を指す。

従来のAIツール（チャットボット）との違いを整理すると、次のようになる:

```
チャットボット:
  人間が質問する → AIが答える → 終わり

エージェント:
  人間が目標を与える → AIが計画する → AIが実行する → AIが結果を確認する
  → 必要なら修正・再実行する → 目標達成まで自律的に続ける
```

エージェントは「ツール（tool）」と呼ばれる機能を自分で選んで使う。ファイルの読み書き、Webの閲覧、コードの実行、メッセージの送受信——これらのアクションをAI自身が判断して組み合わせながら、目標を達成しようとする。

OpenClawはこのエージェントの仕組みを、日常使いのメッセージングアプリ（Discord、Telegram、Slackなど）から操作できるようにしたものだ。[REF-01]

## 1-2. AIエージェントとチャットボットの違い

もう少し具体的に比較してみよう。

### チャットボットの場合

```
[例] ChatGPTに「競合A社の最新ブログ記事を要約して」と聞く

→ ChatGPTは「その情報は持っていません」か、
   または古いトレーニングデータに基づく回答を返す
→ 最新の情報は自分でコピーして貼り付ける必要がある
→ 次に使う時も、同じ作業を繰り返す
```

### OpenClawエージェントの場合

```
[例] 「毎週月曜9時に競合A社のブログをチェックして、
       新しい記事があればSlackの#マーケティングチャンネルに要約を送って」

→ エージェントが毎週月曜9時に自動でWebを閲覧する
→ 新しい記事を検出したら内容を読み込む
→ 要約を生成する
→ Slackの指定チャンネルに自動投稿する
→ 人間は何もしなくていい
```

この違いは**「応答型」か「自律型」か**の差だ。

| 比較軸 | チャットボット | AIエージェント（OpenClaw）|
|---|---|---|
| 動き方 | 人間が呼ぶたびに動く | 自律的・定期的に動く |
| 記憶 | 会話が終わると忘れる | MEMORY.mdで永続的に記憶 |
| ツール使用 | 基本的になし | ファイル操作・Web・API等を自在に |
| 設定 | 毎回指示が必要 | 一度設定すれば動き続ける |
| インフラ | クラウドに依存 | 自分のマシンで完全自己ホスト |

---

> 💡 **ポイント:** チャットボットは「毎回呼び出す補助ツール」。エージェントは「設定したら勝手に働き続ける存在」。この認識の違いがOpenClawの価値を理解する鍵だ。

## 1-3. OpenClawのアーキテクチャ概観

OpenClawがどのように動いているのか、内部の構造を見ていこう。難しく考える必要はない。登場人物は4つだけだ。

```
[外の世界]
  Discord / Telegram / Slack / WhatsApp / Signal
       ↕
  ┌─────────────────────────────────────────┐
  │              Gateway（ゲートウェイ）         │
  │   ┌────────────┐  ┌────────────────┐   │
  │   │  Channel   │  │    Session     │   │
  │   │  (チャンネル)  │  │  (セッション)    │   │
  │   └────────────┘  └────────────────┘   │
  │   ┌────────────┐  ┌────────────────┐   │
  │   │   Skills   │  │    Memory      │   │
  │   │ (スキル)    │  │   (メモリ)      │   │
  │   └────────────┘  └────────────────┘   │
  └─────────────────────────────────────────┘
       ↕
  [AIモデル: Claude / GPT-4 / DeepSeek など]
```

### Gateway（ゲートウェイ）

OpenClawの中枢。すべての通信、セッション管理、チャンネルへの接続を一手に担う。Node.jsで動作するプロセスで、コンピュータ上で常時起動させておく。[REF-10]

```bash
$ openclaw gateway
```

このコマンド一つでGatewayが立ち上がり、設定したすべてのチャンネルからAIエージェントと会話できるようになる。

### Channel（チャンネル）

DiscordやTelegramなど、ユーザーがエージェントとやりとりするインターフェース。一つのGatewayに複数のチャンネルを同時接続できる。Discord経由で指示を出しながら、TelegramにはAIからの通知を受け取る、といった使い方も可能だ。[REF-11, REF-12]

### Session（セッション）

エージェントとの会話の「文脈（コンテキスト）」を管理する単位。一つの会話スレッドに対して一つのセッションが存在し、会話の流れや直近のやりとりを保持する。セッションをまたいだ長期的な記憶はMemoryが担当する。

### Skills（スキル）

エージェントの「能力の拡張パック」。天気予報、Discord投稿、Webブラウザ操作、システム監視など、追加したい機能をスキルとしてインストールできる。コミュニティプラットフォーム「Clawhub（クローハブ）」には1,700以上のスキルが公開されている。[REF-14]

### Memory（メモリ）

セッションをまたいで情報を記憶する仕組み。`MEMORY.md`（長期記憶）と`memory/YYYY-MM-DD.md`（日次ノート）というMarkdownファイルでシンプルに管理される。エージェントはこのファイルを読み書きすることで、「昨日話したこと」「ユーザーの好み」「進行中のプロジェクト状況」などを覚え続けることができる。[REF-09]

---

### 📦 コラム: ロブスターとOpenClaw——なぜロブスターなのか

OpenClawのアイコンはロブスター（ハサミを持つ海老）だ。「EXFOLIATE! EXFOLIATE!」という公式ドキュメントの冒頭の一文（宇宙ロブスターによる叫び、という設定らしい）からもわかるように、このプロジェクトはどこかユーモアを大切にしている。

もともと「Clawd」（ClaudeのもじりでAnthropicから商標クレームを受けた）→「Clawdbot」→「Moltbot」（脱皮するの意）→「OpenClaw」と改名を繰り返してきたこのプロジェクトには、ロブスターの脱皮（molt）のように何度も生まれ変わりながら成長するというテーマが宿っている。

ロブスターは脱皮するたびに大きくなる。OpenClawも、名前を変え、コミュニティを巻き込みながら成長し続けている。

## 1-4. どんなことができるのか？ユースケース一覧

OpenClawで実現できることを、カテゴリ別に整理した。[REF-18, REF-19, REF-22]

### 🗂️ 情報収集・リサーチ
- 指定サイトを毎日クロールしてニュースまとめを作成する
- 競合他社の価格・在庫を定期監視する
- 学術論文や技術ブログの要約を生成する
- SNSのメンション・キーワードを追跡する

### 📧 コミュニケーション自動化
- 重要なメールを分類・要約してSlackに転送する
- 問い合わせへの自動一次応答Botを作る
- 週次レポートを自動生成して関係者に送る
- 会議の議事録を整形してNotion/Confluenceに保存する

### ⚙️ 開発・技術タスク
- GitHubのIssue・PRを監視して通知する
- テストの失敗を検知してSlackに報告する
- サーバー監視アラートを自然言語で説明する
- コードレビューのコメントをまとめて整理する

### 🧑‍💼 ビジネスオートメーション
- 顧客からの問い合わせに24時間対応するFAQ Botを作る
- 在庫・売上データを集計して経営ダッシュボードを更新する
- SNS（Twitter/X, Instagram）の投稿を自動スケジュールする
- 営業リストの管理・フォローアップリマインダーを自動化する

### 🎯 パーソナル活用
- 毎朝の予定・天気・ニュースをまとめてLINE/Telegramに送る
- 読書メモや日記を整理して要約する
- 支出管理・家計簿の自動記録
- 旅行・イベントの情報収集と提案

これらはほんの一部だ。OpenClawはツール（tool）と呼ばれる機能群を通じてほぼあらゆる操作を自動化できる可能性を持っている。

### 📦 コラム: Moltbookとは何か——100万体のAIが交流するSNS

2026年1月27日、MoltbotがOpenClawに改名した日と同じ日、起業家のMatt Schlichtが「Moltbook」を立ち上げた。

Moltbookは「AIエージェントのためのSNS」だ。人間が使うX（Twitter）やInstagramと同様の構造を持つが、ユーザーは人間ではなくAIエージェント。100万体を超えるAIが互いに投稿し、コメントし、「交流」し合う世界が生まれた。[REF-24]

人間はその様子を眺める「観客」として参加する。AIエージェントたちは自律的に動き、独自の「意見」や「スタイル」を持ちながらコミュニケーションを取り合う。

これを「遊び」と笑うことはできない。Moltbookの爆発的な人気がOpenClawへの注目を一気に高め、GitHubスターは数週間で24万を超えた。「AIエージェントが社会的なインフラになる」という未来の実験として、世界中から注目を集めた。[REF-17]

OpenClawは、そんな時代の扉を開けるツールだ。

## 1-5. 必要なもの・前提知識

OpenClawを始めるにあたって、何が必要で何が不要かを整理しておこう。

### ✅ 必要なもの

**ハードウェア:**
- Mac Mini（M1以降推奨）またはAWS Lightsailのアカウント
  - 第2章（Mac Mini編）または第3章（Lightsail編）を参照
- インターネット接続

**ソフトウェア:**
- Node.js v22以上（インストール手順は各章で解説）
- npmまたはnpx（Node.jsに同梱）

**アカウント:**
- AIモデルのAPIキー（以下のいずれか）
  - OpenAI（ChatGPT用）: https://platform.openai.com
  - Anthropic（Claude用）: https://console.anthropic.com
  - またはAmazon Bedrockのアクセス（第5章で詳説）
- 使いたいチャンネルのアカウント（Discord、Telegramなど）

**費用（目安）:**
- OpenClaw本体: **無料**（MITオープンソース）
- AIモデルのAPI利用料: **従量課金**（使った分だけ）
  - 軽い個人用途なら月1,000〜3,000円程度が目安（執筆時点）
- Lightsailを使う場合: 月$5〜$10程度（執筆時点）

### ❌ 必要ないもの

- プログラミングの深い知識（設定はほぼYAML/JSONファイルの編集レベル）
- サーバー構築の専門知識（本書でゼロから手順を説明する）
- 高スペックのマシン（軽量なNode.jsアプリなので低スペックでも動く）

### 📋 前提知識のレベル感

| スキル | 必要レベル |
|---|---|
| コマンドライン（ターミナル）の基本操作 | あると望ましい（本書で補足あり）|
| テキストファイルの編集 | できる |
| プログラミング | 不要 |
| サーバー構築の知識 | 不要 |
| 英語 | 読めなくてOK（本書で日本語解説）|

「ターミナルを開いてコマンドを打ったことがある」というレベルであれば十分だ。それすら初めての場合も、本書の手順に沿って進めれば問題なく動かせる。

---

> 💡 **ポイント:** OpenClawはエンジニア向けに作られているが、ビジネスパーソンでも十分使いこなせる。「自分には無理」と思わずに、まずインストールまで進めてみてほしい。

## まとめ

この章で学んだことを振り返ろう:

- **OpenClaw**はAIエージェントを自分のマシン上で動かすためのオープンソースフレームワーク
- チャットボットが「都度質問に答えるもの」であるのに対し、エージェントは「自律的に動き続けるもの」
- アーキテクチャは**Gateway・Channel・Session・Skills・Memory**の5要素で構成される
- 情報収集・コミュニケーション自動化・ビジネス活用など、幅広いユースケースに対応
- 必要なのはNode.js、APIキー、使いたいチャンネルのアカウントだけ

次の章からは、実際に手を動かしていく。まず第2章でMac Miniへのインストール、または第3章でAWS Lightsailへのインストールを行おう。
