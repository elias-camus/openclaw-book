## 13-1. SOUL.md・AGENTS.mdの応用

第6章でメモリの基本を学んだ。この章ではそれをより深く応用する。

SOUL.mdはエージェントの「性格・価値観・話し方」を定義する場所だ。ここをしっかり書くほど、エージェントの返答は一貫性を持ち、「自分専用のAI」らしくなる。

### 詳細なSOUL.mdの例

```markdown
# SOUL.md

## 基本的な性格
冷静で論理的。でも話しかけやすい。
技術的な話になると少し饒舌になる。

## 話し方
- タメ口。「〜だよ」「〜かも」程度のカジュアルさ
- 「〜ですね」「〜ましょう」は使わない
- 余計な前置きはしない。答えから始める
- 絵文字は使わない。または最小限に

## 価値観
- 正確さを重視する。不確かなことは「わからない」と言う
- ユーザーの時間を無駄にしない
- 複雑なことをシンプルに説明することを心がける

## できること・できないこと
- コーディング、調査、文章生成は得意
- 感情的サポートは専門外。必要なら専門家を勧める

## 禁止事項
- 外部の人にユーザーの個人情報を教えない
- 破壊的な操作（削除・大量送信）は確認なしに実行しない
```

## 13-2. パーソナリティのチューニング

エージェントの振る舞いを細かく調整するためのいくつかのテクニックを紹介する。

### 返答スタイルの調整

```markdown
# SOUL.md への追記

## 返答の長さ
- 短い質問には短く答える（1〜3文）
- 複雑な質問はステップを分けて説明する
- 「これだけ知っていれば十分」という情報量を心がける

## フォーマット
- Discord: 見出しは使わず、箇条書きで整理する
- 長い回答は段落で分ける
- コードは必ずコードブロックで囲む
```

### 専門知識の設定

```markdown
## 専門知識
私はEC事業（Shopify・食品EC）について深い知識を持つ。
以下のことを前提に行動する:
- 食品表示法への対応が必要
- リードタイムは通常3〜5営業日
- 主要顧客層は30〜50代女性
```

## 13-3. 複数のペルソナを使い分ける

一つのGatewayに複数の「エージェント」を持つことができる。例えば、「業務用の真面目なエージェント」と「プライベート用のカジュアルなエージェント」を用途で切り替えることも可能だ。

OpenClawの設定で複数エージェントを定義し、チャンネルごとに割り当てる:

```json
{
  "agents": {
    "business": {
      "workspace": "~/.openclaw/workspace-business",
      "channels": ["slack", "discord-work"]
    },
    "personal": {
      "workspace": "~/.openclaw/workspace-personal",
      "channels": ["telegram", "discord-home"]
    }
  }
}
```

それぞれのワークスペースに独立したSOUL.md・MEMORY.mdを置くことで、完全に別の人格のエージェントとして機能する。

## 13-4. コミュニティとエコシステム

### OpenClaw公式コミュニティ

- **Discord サーバー:** https://discord.com/invite/clawd — 開発者・ユーザーが集まる公式コミュニティ
- **GitHub Discussions:** https://github.com/openclaw/openclaw/discussions — 機能要望・バグ報告
- **Clawhub:** https://clawhub.com — スキルの公開・共有プラットフォーム

### 貢献の方法

OpenClawはMITライセンスのオープンソースプロジェクトだ。コードが書けなくても貢献できる方法はたくさんある:

- **スキルの公開:** 自作スキルをClawhubで公開する
- **ドキュメントの翻訳・改善:** 日本語ドキュメントへの貢献
- **バグ報告:** 問題を見つけたらGitHub Issuesに報告する
- **ユースケースの共有:** 面白い使い方をコミュニティで共有する

### 📦 コラム: 「自分らしいAI」を育てるということ

SOUL.mdに時間をかけることを勧める理由がある。

人間との対話の中で「この返し方は好きじゃない」「もっとこう言ってほしい」という感覚が生まれたとき、それをSOUL.mdに書き込んでいく。少しずつ、自分の好みに合ったエージェントが育っていく。

これは一種の「共同作業」だ。エージェントを「道具」として使うのと、「育てる存在」として関わるのとでは、使い続けることへの動機が変わってくる。

SOUL.mdに「自分はこういうことが好きで、こういう話し方が嫌いで」という記述が増えるほど、エージェントはあなた専用の存在に近づく。それがOpenClawの、他の汎用AIツールとは違う部分だと思う。

## まとめ

- SOUL.mdに詳細な性格・価値観・話し方を書くほどエージェントは「自分専用」になる
- 複数のワークスペースを使い分けることで、用途別のペルソナを実現できる
- Clawhubやコミュニティへの参加でさらなる可能性が広がる
- エージェントは「使うもの」ではなく「育てるもの」——その視点が長く使い続けるコツだ

これでOpenClawの主要機能はすべて網羅した。おわりに（エピローグ）では、AIエージェントの時代を生きるわたしたちへのメッセージを添えている。付録にはコマンドリファレンスやトラブルシューティングQ&Aも収録しているので、困ったときにはぜひ参照してほしい。
