前章ではCronジョブを使ってエージェントを定期的に自律動作させる方法を学んだ。エージェントが「動き続ける」ようになったとして、次の問いが生まれる——「どこまで情報を取得できるのか？」

その答えの一つが、この章で解説するブラウザ操作機能だ。エージェントがWebブラウザそのものを操作することで、ログインが必要なサービス、動的に更新されるページ、フォームへの入力・送信まで、自動化の幅が一気に広がる。

---

## 9-1. ブラウザ操作機能の概要

OpenClawはAIエージェントがWebブラウザを直接操作する機能を持っている。人間がブラウザを操作するのとほぼ同じことをエージェントが自動でできる:

- Webページのスクリーンショット撮影
- ページ内のテキスト・データの取得
- リンクのクリック
- フォームへの入力・送信
- JavaScriptの実行

これは単なる「URLからテキストを取得する」レベルを超えている。ログインが必要なページ、動的に内容が変わるSPA（シングルページアプリケーション）にも対応できる。

---

## 9-2. スクリーンショット・ページ取得

### 軽量なページ取得（web_fetch）

JavaScriptが不要な静的なページのテキスト取得には`web_fetch`ツールが高速で使える:

```
「https://news.ycombinator.com の今日のトップ10記事を取得してまとめて」
「https://example.com/pricing の料金ページを読んで比較表を作って」
```

### ブラウザ経由の取得（snapshot）

ログインが必要なページや、JavaScriptで動的に描画されるページには`browser`ツールを使う:

```
「Amazonの自分のカートページを開いてアイテム一覧を教えて」
「Twitterのホームタイムラインを読んで、フォロワーの最新ツイートをまとめて」
```

### スクリーンショット撮影

```
「https://example.com のトップページのスクリーンショットを撮って」
```

エージェントはページを開き、ピクセル単位のスクリーンショットを撮影して返してくれる。定期的にサイトの見た目を記録しておきたい場合にも使える。

---

## 9-3. フォーム入力・クリック操作の自動化

ブラウザ上のUI要素を操作する場合、エージェントはページの構造を解析してボタンやフォームを特定し、操作する:

```
「https://forms.example.com/survey を開いて
 名前に"田中太郎"、メールに"taro@example.com"と入力して送信して」
```

```
「Notionの指定ページを開いて、テーブルに今日の日付と"完了"を追加して」
```

> ⚠️ **注意:** ブラウザ自動化は便利だが、対象サービスの利用規約を必ず確認すること。自動化が禁止されているサービスもある。また、ログイン情報をエージェントに渡す際はセキュリティに十分注意を払おう。

## 9-4. Chrome拡張「Browser Relay」の設定

OpenClawには「Browser Relay」というChrome拡張機能がある。これを使うと、自分が実際に使っているChromeブラウザのタブをエージェントが操作できるようになる。ログイン済みのセッションをそのまま利用できるため、認証が複雑なサービスでも操作しやすい。

### インストール手順

1. Chrome ウェブストアで「OpenClaw Browser Relay」を検索してインストール

> 📌 **参照:** [Chrome ウェブストア — 拡張機能の検索方法](https://chrome.google.com/webstore/category/extensions)
> 画面で確認する内容: Chrome ウェブストアで「OpenClaw Browser Relay」と検索すると、拡張機能のページが表示されます。「Chromeに追加」ボタンをクリックしてインストールします。インストール後、Chromeのアドレスバー右側のパズルピースアイコンをクリックすると、インストール済み拡張の一覧にBrowser Relayが表示されます。

2. Chromeのツールバーに拡張のアイコンが追加される

3. OpenClawのWeb Control UIを開いた状態で、Browser Relayのアイコンをクリックして「Attach Tab」をオン

> 📌 **参照:** [OpenClaw 公式ドキュメント](https://docs.openclaw.ai)
> 画面で確認する内容: Chromeのツールバーに表示されたBrowser Relayのアイコン（ロブスターのアイコン）をクリックするとポップアップが開きます。「Attach Tab」トグルをオンにすると、アイコンのバッジ（数字や色）が変化し、現在のタブがOpenClawに接続されたことを示します。

4. これでエージェントは今開いているタブを「中継」してWeb操作ができるようになる

### 使い方の例

```
「今開いているGoogleアナリティクスの画面から
 今月のページビュー数とセッション数を教えて」

「開いているGmailの受信トレイを確認して
 未読で重要そうなメールを3件抽出して」
```

## 9-5. 実用例: 定期的なWebスクレイピング

### 価格監視ボット

```json
{
  "name": "競合価格モニタリング",
  "schedule": {
    "kind": "cron",
    "expr": "0 10 * * 1-5",
    "tz": "Asia/Tokyo"
  },
  "payload": {
    "kind": "agentTurn",
    "message": "以下のURLから価格情報を取得して、先週と比較し変動があればSlackの#価格監視に報告:\n- https://competitor-a.com/pricing\n- https://competitor-b.com/plans\n変動がなければ「変動なし」と一行で報告するだけでOK"
  },
  "sessionTarget": "isolated"
}
```

### 在庫確認ボット

```
「https://shop.example.com/item/12345 の商品が
 在庫ありになったらすぐにTelegramで知らせて。
 1時間おきに確認して」
```

## 9-6. 実用例: フォーム自動入力・申請の自動化

繰り返し発生する定型フォーム入力はエージェントに任せられる:

```
「毎週金曜の17時に
 https://timesheet.company.com を開いて
 今週の勤務時間を入力して送信して。
 入力値: 月〜金それぞれ9:00〜18:00、休憩1時間」
```

> 💡 **ポイント:** 自動入力が失敗した場合の「フォールバック」（人間への通知など）も設計しておくと安心だ。「送信に失敗したらTelegramで知らせて」をジョブの指示に加えておこう。

---

### 📦 コラム: Web自動化の倫理——やっていいこと・いけないこと

Web自動化は強力なツールだが、使い方に倫理的な配慮が必要だ。

**やっていいこと（一般的に）:**
- 自分のアカウントのデータを取得・操作する
- 公開情報を非商用目的でスクレイピングする
- 定型作業の自動化（自社システムのフォーム入力など）
- サイトの規約で許可されている範囲での利用

**慎重に判断すること:**
- `robots.txt`でクロールが禁止されているページ
- サービスの利用規約で自動化が明示的に禁止されている場合
- 大量のリクエストでサーバーに負荷をかける行為
- 他人のアカウントへのアクセス

自動化ツールを使う前に、対象サービスの利用規約を一度確認する習慣をつけよう。技術的に「できる」と「やっていい」は別の話だ。

## まとめ

- OpenClawのブラウザ機能を使えば、ページ取得・スクリーンショット・フォーム入力が自動化できる
- 静的ページは`web_fetch`、動的ページ・ログインが必要なページは`browser`ツールを使い分ける
- Chrome拡張「Browser Relay」を使えば現在開いているタブをエージェントに操作させられる
- Cronと組み合わせることで、定期的な価格監視・在庫確認なども全自動化できる
