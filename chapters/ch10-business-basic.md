## 10-1. メール・メッセージの自動応答

OpenClawをメールシステムに接続すると、受信メールをリアルタイムで処理できる。Gmail連携にはGmail PubSub（パブサブ）を使う。[REF-01]

### Gmailとの連携概要

Gmail → Cloud Pub/Sub → OpenClaw Webhook → エージェント処理 → 返信

```
「重要度の高いメールが来たらすぐに要約してSlackで知らせて。
 返信が必要そうなら下書きを作って教えて」
```

### 自動応答の設計指針

いきなり全自動で返信させるのはリスクがある。段階的な設計を推奨する:

**ステップ1（最初）:** メールを分類・要約して通知するだけ  
**ステップ2（慣れたら）:** 定型的な問い合わせは下書きを生成して確認を求める  
**ステップ3（信頼できたら）:** 特定カテゴリのみ自動返信を許可  

> 💡 **ポイント:** 「自動返信」は便利だが、誤った内容の返信が相手に届くリスクがある。最初は人間が確認するステップを必ず挟もう。

## 10-2. 議事録・要約の自動生成

会議の録音や文字起こしテキストをエージェントに渡すと、構造化された議事録を自動生成できる。

### フロー例

```
Zoom/Meet の文字起こし → ファイルとしてエージェントに渡す
→ エージェントが整形（決定事項・ToDo・次回日程を抽出）
→ Slackの #議事録 チャンネルに投稿
→ Notionの指定ページにも保存
```

### プロンプト例

```
「添付の会議録音を文字起こしして以下の形式で議事録を作成して:
## 日時・参加者
## 議題ごとの議論サマリー
## 決定事項（箇条書き）
## アクションアイテム（担当者・期限付き）
## 次回会議の予定

完成したらSlackの#議事録に投稿して」
```

## 10-3. データ収集レポートの自動化

定期的に複数ソースからデータを集めてレポートを作るタスクは、Cronと組み合わせると完全自動化できる。

```json
{
  "name": "週次競合レポート",
  "schedule": { "kind": "cron", "expr": "0 9 * * 1", "tz": "Asia/Tokyo" },
  "payload": {
    "kind": "agentTurn",
    "message": "毎週月曜のレポートを作成して:\n1. 競合A社・B社のブログ新着記事（web_fetchで取得）\n2. 業界キーワードのGoogle Trendsの動向\n3. 自社製品に関するSNSメンション数（可能であれば）\nMarkdown形式でまとめてSlackの#weekly-reportに投稿",
    "timeoutSeconds": 300
  },
  "sessionTarget": "isolated"
}
```

## 10-4. カスタマーサポートBotの構築

DiscordやSlackに顧客対応Botを設置する基本パターン。

### SOUL.mdでサポート担当者の人格を定義

```markdown
# SOUL.md

あなたは株式会社〇〇のカスタマーサポート担当「サクラ」です。

## キャラクター
- 丁寧で親切。でも堅すぎない。
- 問題解決を最優先に考える
- 解決できないときは素直に「担当者に確認します」と伝える

## 対応方針
- まず問題の内容を確認する
- FAQデータベース（MEMORY.mdに記載）を参照する
- 解決できない場合はsupport@example.comへのエスカレーションを案内する
```

### FAQをMEMORY.mdに書いておく

```markdown
# FAQ（カスタマーサポート用）

## よくある質問

**Q: パスワードを忘れた**
A: ログインページの「パスワードをお忘れの方」からリセットメールを送れます。

**Q: 返金ポリシーは？**
A: 購入後30日以内であれば全額返金対応しています。support@example.com へご連絡ください。

**Q: 対応時間は？**
A: 平日9:00〜18:00です。時間外のご連絡は翌営業日に対応します。
```

## 10-5. 社内FAQボットの作り方

SlackやDiscordに社内向けのFAQ Botを設置すると、新入社員の質問対応や社内ルール確認を自動化できる。

**構成のポイント:**
1. 社内ルール・よくある質問をMEMORY.mdにまとめる
2. SOUL.mdで「社内担当者」としての口調を定義する
3. Slackの特定チャンネル（例: `#新入社員`）にBotを設置
4. 答えられない質問は「人事部に確認します」と答えてエスカレーション

```
 各案はタイトル・見出し構成・300字の概要つきで
 Notionの下書きDBに追加して」
```

**課題:** 予約問い合わせ・メニュー質問への対応が営業時間外にできない

```
 スタッフ"あかね"です。明るく親しみやすく対応してください」
```

### 💼 フリーランス・個人事業主

**課題:** 見積もり対応・スケジュール調整に時間がかかる

**OpenClaw活用例:**
- 問い合わせメールに自動で「受け取りました、〇〇営業日以内に返答します」を送信
- カレンダーを確認して空き日程を自動で案内

### 🛒 ECショップ

**課題:** 在庫管理・価格変動への対応が手動で追いつかない

**OpenClaw活用例:**
- 競合ECサイトの価格を毎日チェックして価格差レポートを作成

- カスタマーレビューを週次で収集・ポジネガ分析してレポート化

### 📊 コンサルタント・士業

**課題:** リサーチと報告書作成に工数がかかりすぎる

**OpenClaw活用例:**
- クライアントの業界ニュースを毎日収集してブリーフィングシート作成
- 会議の録音から議事録・アクションアイテムを自動生成
- 月次報告書のデータ収集・初稿作成を自動化

---

### 🏠 不動産

**課題:** 物件問い合わせの初期対応が24時間必要

**OpenClaw活用例:**
- 物件情報をMEMORY.mdに整理し、LINEからの問い合わせに自動応答
- 「2LDK 渋谷近辺 予算7万円以内」といった条件に合う物件を即座に提案
- 内覧のスケジュール調整を自動化

### 📦 コラム: スモールビジネスオーナーのOpenClaw活用——ある美容院の事例

（架空の事例として）

東京・中目黒でオーナー一人で営む小さな美容院。Instagram DMへの問い合わせ対応が悩みだった。営業中は手が離せず、深夜に返信しても相手が起きていない。

OpenClawを導入してInstagram DMにBotを接続した。営業時間外の問い合わせには自動で返答する:

「こんにちは！翌日以降のご予約はこちらのURLからどうぞ→[予約URL]。ご質問があれば承ります😊」

空席状況はGoogleカレンダーと連動させ、「今週土曜の午後は空いていますか？」という質問にも自動で答えられるようにした。

導入後、深夜の作業が週に3〜4時間削減された。その時間を接客の質向上に使えるようになったという。

---

## まとめ

- メール自動応答は段階的に（通知→下書き→全自動の順）導入するのが安全
- カスタマーサポートBotはSOUL.mdとMEMORY.mdを組み合わせて設計する
- 飲食・EC・フリーランス・コンサルなど、業種を問わずスモールビジネスに活用できる
- 「AIが24時間対応している」という状態が、小さなビジネスの競争力を大きく変える
