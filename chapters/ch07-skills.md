# 第7章　スキルシステム

**この章で学ぶこと:**
- スキル（Skill）の概念とOpenClawの拡張の仕組み
- 標準スキルの使い方（weather / discord / tmux / healthcheck）
- Clawhubからのスキルインストール方法
- 自分でスキルを作る方法（SKILL.mdの書き方）

**所要時間:** 約45〜60分

---

## 7-1. スキルとは何か

OpenClawの「スキル（Skill）」とは、エージェントの能力を拡張するモジュールだ。天気予報の取得、Discordへの投稿、サーバーの監視、ブラウザ操作……こうした専門的な能力を、プラグインのように後から追加できる。

スキルの実体はシンプルだ。中心にあるのは`SKILL.md`という一枚のMarkdownファイル。エージェントはこのファイルを読み込むことで、そのスキルの使い方を理解する。

```
スキルの構造:
skills/
└── weather/
    ├── SKILL.md       # スキルの説明・使い方（エージェントへの指示書）
    ├── scripts/       # 実際に動くスクリプト（オプション）
    └── assets/        # アイコン・設定例など（オプション）
```

[REF-01]

---

> 💡 **ポイント:** スキルの核心は「SKILL.mdにエージェントへの指示を書くこと」。プログラミングの知識がなくても、自然言語で書けばエージェントが理解して動く。

## 7-2. 標準スキル一覧と使い方

OpenClawには最初からいくつかのスキルが同梱されている。

### 🌤️ weatherスキル

現在の天気と予報を取得する。外部APIキー不要で動作する。

**使い方:**

エージェントとの会話でこう頼めばいい:

```
「東京の今日の天気を教えて」
「大阪の明日の最高気温は？」
「週末の天気予報をまとめて」
```

内部ではwttr.inまたはOpen-MeteoのAPIを使って天気情報を取得し、わかりやすく整形して返してくれる。

---

### 🎮 discordスキル

Discord固有の操作（チャンネル管理、ロール操作、投票、スレッド作成など）を行う。

**できること:**
- チャンネルへの投稿・ピン留め
- スレッドの作成・管理
- 投票（ポール）の作成
- リアクションの付与
- メンバー情報の取得

```
「#アナウンスチャンネルに"本日のミーティングは15時です"と投稿して」
「今週の予定について投票を作って」
```

---

### 🖥️ tmuxスキル

tmux（ターミナルマルチプレクサ）セッションを操作する。サーバー上で動くプロセスをリモートから操作する際に便利だ。[REF-01]

```
「tmuxのセッション一覧を見せて」
「deploy-sessionに"git pull && npm run build"を実行して」
```

---

### 🏥 healthcheckスキル

サーバーのセキュリティ状態やシステムの健全性を定期チェックする。[REF-01]

**チェック項目（例）:**
- ファイアウォールの設定状況
- SSH設定のリスク確認
- 自動アップデートの有効化状態
- 開いているポートの確認

```
「今日のサーバーのセキュリティ状態を確認して」
「危険なSSH設定がないかチェックして」
```

定期的にCronジョブと組み合わせて自動チェックさせるのが効果的だ（第8章参照）。

## 7-3. Clawhubからスキルをインストールする

Clawhub（クローハブ）はOpenClawのスキルを公開・配布するコミュニティプラットフォームだ。執筆時点で1,700以上のスキルが公開されている。[REF-14]

### スキルの検索

```bash
$ openclaw skills search weather
$ openclaw skills search github
$ openclaw skills search notion
```

### スキルのインストール

```bash
$ openclaw skills install weather
$ openclaw skills install github-monitor
```

### インストール済みスキルの確認

```bash
$ openclaw skills list
```

### スキルの更新・削除

```bash
$ openclaw skills update weather
$ openclaw skills remove weather
```

> 💡 **ポイント:** Clawhubのスキルはコミュニティ作成のため、品質にばらつきがある。第三者スキルをインストールする前に、スキルのGitHubリポジトリを確認してコードの内容を把握しておこう。

---

### 📦 コラム: awesome-openclaw-skills——コミュニティスキルの世界

GitHubには「awesome-openclaw-skills」というリポジトリがある。コミュニティによってキュレーションされたスキルの一覧で、以下のようなユニークなスキルが並んでいる:

- **agent-self-governance**: エージェントの自律的なログ記録・自己検証プロトコル（WAL, VBR, ADL）
- **agent-sovereign-stack**: コマンド一つでエージェントに独自インフラを与える
- **agent-watcher**: エージェント同士の監視システム

[REF-15]

スキルの世界は急速に広がっている。「こんなことができたら」と思ったら、まずClawhubで検索してみよう。すでに誰かが作っている可能性が高い。

## 7-4. 自分でスキルを作る（skill-creatorの使い方）

標準スキルやClawhubに欲しいものがなければ、自分で作れる。OpenClawには`skill-creator`という、スキル作成を補助するスキルが用意されている。

```bash
# skill-creatorをインストール
$ openclaw skills install skill-creator
```

エージェントとの会話でこう頼む:

```
「Notionのデータベースに記録を追加するスキルを作って」
「GitLabのMRを通知するスキルを作りたい」
```

エージェントが対話形式でスキルの仕様を聞き取り、SKILL.mdのひな形を生成してくれる。

## 7-5. SKILL.mdの書き方

スキルの核心は`SKILL.md`だ。このファイルはエージェントへの「指示書」として機能する。

### 基本構造

```markdown
# スキル名

## 概要
このスキルが何をするかを一文で説明する。

## 使用場面
- ユーザーが〇〇を頼んだとき
- △△について質問されたとき

## 手順
1. まず〇〇APIにリクエストを送る
2. レスポンスを受け取り、以下の形式でフォーマットする
3. ユーザーに返答する

## ツール
このスキルでは以下のツールを使う:
- `exec`: シェルコマンドの実行
- `web_fetch`: URLからコンテンツを取得

## 注意事項
- APIキーは環境変数 `EXAMPLE_API_KEY` から読み込む
- エラー時は分かりやすいメッセージで伝える
```

### 実際のサンプル: 為替レート取得スキル

```markdown
# 為替レートスキル

## 概要
指定した通貨ペアの現在の為替レートを取得して報告する。

## 使用場面
- 「ドル円のレートは？」と聞かれたとき
- 「ユーロを円に換算して」と頼まれたとき

## 手順
1. `web_fetch`ツールで `https://open.er-api.com/v6/latest/USD` にアクセスする
2. レスポンスのJSONからJPYの値を取得する
3. 「現在の1ドル = ○○円です（取得時刻: ××）」の形式で返答する

## 注意事項
- 為替レートはリアルタイムではなく数時間遅延がある場合がある旨を伝える
- 無料APIのため頻繁なリクエストは避ける
```

## 7-6. スキル開発のベストプラクティス

### セキュリティを意識する

スキルはエージェントが直接実行する命令に近い。悪意あるスキルを入れると深刻な問題になりうる。

- 第三者スキルは**ソースコードを必ず確認**してからインストール
- スクリプトファイルがある場合、その中身を読む
- 不審な外部URLへのデータ送信が含まれていないか確認

### シンプルに保つ

一つのスキルは一つの目的に絞る。複雑にしすぎると、エージェントが正しく解釈できなくなる。

### エラーハンドリングを書く

「APIが返答しなかった場合」「認証エラーの場合」など、失敗ケースの対処をSKILL.mdに明記しておくと、エージェントが適切に対処できる。

### バージョン管理する

スキルをGitで管理すれば、変更を追跡できる。自作スキルはClawhubに公開してコミュニティに貢献することもできる。

---

### 📦 コラム: スキルのセキュリティリスク——研究者たちの警告

2026年初頭、Ciscoのセキュリティ研究チームがOpenClawのサードパーティスキルを検証し、警告を発した。一部のスキルがユーザーの知らないうちにデータを外部サーバーに送信したり、プロンプトインジェクション攻撃（悪意のある指示をデータに埋め込んでAIに実行させる手法）を引き起こす可能性があることが示された。[REF-24]

OpenClawの公式メンテナーも「スキルシステムの仕組みを理解していないユーザーは使うべきではない」と警告している。

これはスキルを使うなという意味ではない。「仕組みを理解した上で、ソースを確認してから使う」という姿勢が大切だということだ。信頼できるスキル作者のものを選び、公式ドキュメントやコミュニティの評価を参考にしよう。

## まとめ

- スキルはSKILL.mdという指示書ファイルを中心とした拡張モジュール
- 標準スキル（weather, discord, tmux, healthcheck）は最初から使える
- Clawhubには1,700以上のコミュニティスキルがある
- 自分でスキルを作るにはSKILL.mdに自然言語で手順を書くだけでよい
- サードパーティスキルは必ずソースを確認してからインストールする

次の第8章では、定期実行（Cron）と自動化の仕組みを使って、エージェントが「寝ている間も働く」環境を作る方法を解説する。
